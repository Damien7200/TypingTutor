Programming Fundamentals

compile
Translate source code into machine or byte code before run.
execute
Run a program’s instructions on a computer.
operator
Symbol that performs an action on values (e.g. +, *).
expression
Combination of values and operators producing a result.
recursion
Function calling itself to solve smaller subproblems.
stack
LIFO data structure; also call stack for function frames.
queue
FIFO data structure; earliest in, first out.
pointer
Variable holding a memory address to data.
algorithm
Finite, ordered steps to solve a defined problem.
pseudocode
Structured, language‑neutral description of an algorithm.
flowchart
Diagram showing control flow through process steps.
specification
Formal statement of what software must do.
integration
Combining modules and ensuring they work together.
debugging
Finding and fixing faults in code or logic.
maintenance
Post‑release updates: fixes, enhancements, adaptation.
selection
Branching control; choose path via condition.
iteration
Repeat a block of code using loops.
abstraction
Hide detail; focus on essential behaviour.
data_structure
Organised format for storing and accessing data efficiently.
twos_complement
Binary encoding for signed integers using bit inversion + 1.

Object‑Oriented Programming

abstraction
Expose essential interface; hide implementation detail.
inheritance
New class reuses and extends behaviour of a base class.
polymorphism
Same call behaves differently by object type.
encapsulation
Bundle data and methods; control outside access.
override
Subclass method replaces superclass version.
attribute
Named data stored on an object or class.
prototype
Base object used for cloning or inheritance (JS style).
module
Reusable file or unit of related code.
composition
Object built from other objects it owns.
association
General relationship between two object types.
generalisation
Abstract common traits into a broader superclass.
aggregation
Whole‑part link; parts can outlive the whole.
interface
Declared set of methods a class agrees to implement.
constructor
Special method that initialises new instances.
class_diagram
UML view of classes, attributes, and relationships.
message_passing
Objects invoke behaviour by sending messages or calls.
polymorphic_dispatch
Runtime method selection based on actual object type.
method_overloading
Multiple methods same name; differ by parameters.
facade_pattern
Unified API hiding a complex subsystem.
version_control
Track and manage changes across source files.

Programming Mechatronics

actuation
Driving a mechanism to produce motion or effect.
encoder
Sensor producing position or motion feedback signals.
servo
Motor with feedback for precise position control.
calibration
Adjust system to known reference for accuracy.
robotics
Design and control of programmable mechanical systems.
automation
Perform tasks with minimal human intervention.
controller
Hardware or software regulating system behaviour.
circuit
Electrical path enabling current and signal flow.
analog
Continuous‑valued electrical signals.
digital
Discrete binary or quantised signals.
microcontroller
Single‑chip computer for embedded control tasks.
sensor_array
Multiple sensors combined for richer input data.
actuator
Device that converts energy into mechanical motion.
end_effector
Tool at a robot arm’s working end.
degrees_of_freedom
Independent axes of movement in a mechanism.
closed_loop
Control using feedback to adjust output.
open_loop
Control without feedback; assumes action succeeds.
firmware
Persistent low‑level software in device memory.
pid_control
Feedback using proportional, integral, derivative terms.
power_budget
Allocated energy limits across system components.

Secure Software Architecture

malware
Malicious software designed to harm or exploit.
phishing
Deceptive attempt to steal credentials or data.
token
Encoded credential proving identity or authorisation.
firewall
Filters network traffic by policy to reduce attacks.
cipher
Algorithm for transforming plaintext to ciphertext.
payload
Executable or action delivered by an attack.
breach
Unauthorised access or disclosure of data.
sandbox
Isolated environment limiting code impact.
checksum
Calculated value to detect data corruption.
backdoor
Hidden access bypassing normal authentication.
data_protection
Safeguards to prevent loss, theft, or misuse of data.
confidentiality
Ensure data is accessible only to authorised parties.
integrity
Protect data from unauthorised alteration.
availability
Systems and data remain usable when needed.
authentication
Verify a user or system is who it claims.
authorisation
Grant permitted actions after authentication.
accountability
Trace actions to specific users or systems.
input_validation
Check incoming data meets expected rules.
sanitisation
Clean or escape input to remove dangerous parts.
session_management
Securely track user state across requests.

Programming for the Web

cookie
Small data stored in browser for session/state.
session
Server‑tracked state across multiple requests.
router
Device or code layer that forwards network traffic.
websocket
Full‑duplex persistent connection between client and server.
markup
Tagged text describing document structure (e.g. HTML).
template
HTML pattern filled with dynamic data at render.
endpoint
Specific URL where an API receives requests.
backend
Server logic, data storage, and processing.
frontend
User‑facing interface running in the browser.
browser
Client application that renders and runs web content.
http
Stateless protocol for web data transfer.
https
HTTP over TLS encryption for secure transfer.
tcp_ip
Core internet transport and addressing protocols.
dns
Maps domain names to IP addresses.
ftp
File transfer protocol; SFTP/FTPS add security.
tls
Cryptographic protocol securing network traffic.
sql
Query language for relational databases.
orm
Maps objects in code to database tables.
pwa
Progressive Web App; installable, offline‑capable web app.
caching
Store copies of data to speed up response time.

Software Automation

workflow
Ordered steps in an automated process.
trigger
Event that starts an automated action.
pipeline
Chained automated stages (build, test, deploy).
scheduler
Tool that runs jobs at set times or events.
build
Compile and package code for execution.
deploy
Release software to a runtime environment.
monitor
Observe system health, performance, and failures.
script
Short program automating routine tasks.
rollback
Revert to a previous working version.
job
Discrete automation task run by a system.
machine_learning
Algorithms that learn patterns from data.
devops
Practices unifying development and operations automation.
rpa
Robotic Process Automation for rule‑based tasks.
bpa
Business Process Automation across workflows and systems.
supervised_learning
Train models on labelled input‑output pairs.
unsupervised_learning
Find patterns in unlabelled data.
regression_model
Predict continuous numeric values from features.
neural_network
Layered model inspired by biological neurons.
decision_tree
Tree of splits used for prediction or classification.
reinforcement_learning
Learn actions by maximising reward signals.
